# 对比实验设计

## 实验目的

验证 GitLearnAgent 是否比“直接把 GitHub 链接交给通用 AI 分析”更适合初学者理解开源项目。

## 实验对象

- 公开 Python 仓库 3 个。
- 公开 React/Vite 或 TypeScript 仓库 3 个。
- 参与者建议为大一或刚学编程的学生 6-12 人。

## 对比方法

每个仓库分别使用两种方式生成导读结果：

1. 直接向通用 AI 提问：请分析这个 GitHub 仓库，告诉初学者如何学习。
2. 使用 GitLearnAgent：输入同一仓库 URL，生成项目地图、学习路线、问答和报告。

## 评价指标

- 模块覆盖率：导读结果覆盖的核心模块数量 / 人工标注核心模块数量。
- 源码引用率：回答中带具体文件路径的比例。
- 幻觉率：出现无法从仓库源码或 README 支撑的说法的比例。
- 学习友好度：参与者对“我知道先看哪里”的 1-5 分评分。
- 完成任务时间：参与者找到入口文件、启动命令、核心模块所用时间。
