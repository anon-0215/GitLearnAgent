# 大创项目申报简稿

## 项目名称

基于静态分析与大模型的开源项目分层导读系统

## 研究背景

低年级学生在学习开源项目时常遇到三个问题：不知道从哪个文件开始读、不理解工程目录的职责划分、直接询问通用 AI 时回答缺少源码依据。GitLearnAgent 将公开 GitHub 仓库自动转化为可学习的项目地图和阶段化任务，帮助初学者以更低成本进入真实工程。

## 项目目标

1. 自动抓取并过滤公开 GitHub 仓库中的文本源码文件。
2. 对 Python、JavaScript、TypeScript 项目进行静态分析，识别依赖、入口、模块和核心文件。
3. 生成面向初学者的学习路线、阅读任务和测验。
4. 支持带源码引用的项目问答。
5. 通过对比实验验证本系统相比直接询问通用 AI 的优势。

## 创新点

- 静态分析与大模型结合：先用规则和代码结构建立可靠上下文，再使用模型生成教学表达。
- 面向初学者的分层学习路径：从全局印象、启动方式、入口文件、模块阅读到小修改任务。
- 可溯源问答：回答必须返回相关文件路径和代码片段。
- 可评测：用模块覆盖率、引用准确率、幻觉率、学生主观评分等指标衡量效果。

## 技术路线

1. GitHub URL 解析与公开仓库文件树抓取。
2. 文本文件过滤，跳过依赖目录、构建产物、二进制文件和大文件。
3. Python 使用 `ast` 提取 import、函数、类；JS/TS 使用轻量正则提取 import/export 和符号。
4. 根据 README、依赖文件、入口文件、源码目录和导入关系计算文件重要性。
5. 按目录和文件职责生成模块地图。
6. 基于结构化分析生成学习路线、任务卡、测验和报告。
7. 问答时先检索源码片段，再调用 OpenAI 兼容模型或规则兜底生成回答。

## 预期成果

- 一个可运行的 Web 系统。
- 5-10 个公开仓库的分析样例。
- 一份项目分析报告导出能力。
- 一组对比实验数据和演示视频。
