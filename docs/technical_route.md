# 技术路线说明

## 1. 抓取层

后端通过 GitHub REST API 获取仓库元信息和递归文件树，再从 `raw.githubusercontent.com` 拉取文本文件。首版只支持公开仓库，可以配置 `GITHUB_TOKEN` 提高请求额度。

## 2. 过滤层

系统跳过 `.git`、`node_modules`、`dist`、`build`、`.venv`、二进制文件和超大文件。这样做可以减少噪声，并避免把依赖库误判为项目源码。

## 3. 静态分析层

- Python：使用标准库 `ast` 提取 import、函数、类和入口线索。
- JS/TS：用轻量正则提取 import/export、函数、类和前端入口线索。
- 配置文件：识别 `package.json`、`requirements.txt`、`pyproject.toml` 等。

## 4. 排序与模块层

文件重要性由 README、依赖文件、入口文件、源码目录、框架标记、导入数量和符号数量共同决定。模块主要按目录和职责聚合，再生成适合初学者阅读的模块解释。

## 5. 教学层

学习路线固定为五步：全局印象、依赖和启动、入口文件、模块阅读、小修改任务。这样即使没有模型，也能生成稳定的导读结果。

## 6. 问答层

问答不会直接让模型猜。系统先检索相关文件和片段，再生成回答。配置了 `LLM_API_KEY` 时使用大模型增强表达；未配置时使用本地规则兜底。
