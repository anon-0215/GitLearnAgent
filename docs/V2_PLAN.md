# GitLearnAgent V2 规划

## 项目定位

GitLearnAgent V2 是面向 Python 开源项目初学者的自适应代码学习系统。它以当前 V1 Demo 为基线，逐步从“仓库导读工具”升级为“可信源码证据驱动的学习工作台”。

## 第一阶段：可信代码学习 MVP

目标：把源码理解的最小可信单元从“文件”推进到“函数和类代码块”，并为后续问答、出题和学习记录建立可追溯证据基础。

范围：

- Python 函数级 AST 分析。
- 函数和类代码块。
- 精确源码行号。
- 内容哈希与缓存基础。
- 本地 BGE-M3。
- 关键词与语义混合检索。
- AST 调用关系扩展。
- 带证据的源码问答。
- 结论分解与证据验证。

验收标准：

- 对一个中小型 Python 仓库，能解析函数、异步函数和类，并保存路径、符号、起止行号、内容和内容哈希。
- 未变化代码块再次分析时不会重复计算 Embedding。
- 关键词检索和语义检索分别保留分数，混合排序结果可追溯到源码文件和行号。
- 问答回答能区分源码事实、静态分析推断和证据不足。
- 回答中引用的文件路径和行号能由程序验证。
- 现有 V1 的仓库抓取、项目概览、学习路线、问答和报告功能仍可运行。

## 第二阶段：自适应教学闭环

目标：在可信代码证据之上，建立面向初学者的诊断、讲解、练习、评分和路线调整闭环。

范围：

- 初始诊断。
- 分层源码讲解。
- 动态学习路径。
- 受约束出题。
- 评分细则。
- 客观题程序评分。
- 开放题模型逐项评分。
- 根据历史表现调整路线。
- 完整学习档案。
- 可追溯学习报告。

验收标准：

- 用户开始学习前能完成一次初始诊断，并生成可保存的能力画像。
- 系统能基于项目代码块和用户水平生成分层讲解。
- 题目生成必须绑定源码证据、难度、目标能力点和评分细则。
- 客观题评分由程序完成，开放题评分必须按 rubrics 分项输出。
- 学习路径会根据用户历史表现调整后续推荐。
- 学习报告能追溯到用户答题记录、源码引用和评分依据。

## 第三阶段：工程完善与实验

目标：完善工程可维护性、模型接口、可视化和实验设计，为大创展示与小规模评测提供稳定支撑。

范围：

- FastAPI、Flask、PyTorch 分析插件。
- DeepSeek 与本地 7B 统一模型接口。
- 仓库 commit 版本和增量更新。
- Prompt 模板版本管理。
- 引导式三栏学习工作台。
- 局部调用关系图。
- Direct LLM、Single Prompt、GitLearnAgent 对照实验。
- 消融实验。
- 小规模用户实验。

验收标准：

- FastAPI、Flask、PyTorch 项目能通过插件扩展识别入口、路由、模型或训练相关结构。
- DeepSeek 兼容 API 和本地 7B 模型通过统一接口调用，并在前端展示模型来源和降级状态。
- 每次分析记录仓库 commit，支持基于 commit 和内容哈希的增量更新。
- Prompt 模板有任务名称、版本、输入 Schema 和输出 Schema。
- 前端提供引导式三栏工作台，能同时查看学习路径、源码证据和问答/练习。
- 至少完成 Direct LLM、Single Prompt、GitLearnAgent 三种方法的对照实验记录。
- 消融实验能分别验证 AST、混合检索、证据验证和教学闭环的贡献。

## 技术选型

- Python AST。
- FastAPI。
- React。
- SQLite。
- BGE-M3。
- DeepSeek 兼容 API。
- 本地 7B 可选备用。
- 本地单机部署。

## 不做范围

第一版不做：

- 完整多语言分析。
- 云端多用户部署。
- 执行第三方仓库代码。
- 自动安装第三方仓库依赖。
- 完整 IDE。
- 图神经网络。
- 强化学习。
- 复杂多智能体系统。
- 大语言模型训练或微调。
- 私有仓库 OAuth 授权。
- 大规模分布式索引服务。
- 自动修复或改写被分析仓库代码。

## 开发顺序建议

1. 函数级 Python 代码块、精确行号、内容哈希与数据库存储。
2. 基于代码块的关键词检索和 BGE-M3 Embedding 缓存。
3. 混合检索和带行号引用的问答。
4. AST 调用关系扩展和证据验证。
5. 诊断、出题、评分和学习档案。

每一步都必须保持现有 Demo 可运行，并记录迁移日志。
